<자료구조와 알고리즘이란?>

1) 자료구조란
	- 용어 : 자료구조, 데이터구조, data structure
	=> 3가지 용어를 혼용해서 사용한다.
	- 대량의 데이터를 효율적으로 관리할 수 있는 데이터의 구조를 의미 한다. 
	=> 현실세계의 있는 정보를 프로그래밍으로 바꾸려면, 정보를 변환하여 데이터로 어떻게 컴퓨터에 저장해야하는가?
		그 구조를 데이터구조라고한다.
	- 코드상에서 효율적으로 데이터를 처리하기 위해, 데이터의 특성에 따라, 체계적으로 데이터를 구조화해야한다.
	=> 어떤 데이터 구조를 사용하느냐에 따라 코드 효율이 달라진다. 

※효율적 데이터 관리의 예 )
	우편번호, 학생관리 (학생부관리) 
	=> 이런 관리 기법이 없다면 특정 하나의 데이터를 찾기위해 어마어마한 데이터를 찾아봐야 한다. 


2) 대표적인 자료구조
	- 배열, 스택, 큐, 링크드 리스트, 해쉬테이블, 힙 등
	=> 대표적인, 프로그래밍에서 가장 많이 사용했던 자료구조들이다.
	이것을 익히면서 새로운 자료구조를 만들고 익힐수 있는 역량을 키울 수있다.
	=> 또한 이런 자료구조가 알고리즘에서 사용되기 때문에, 매우 익히는것이 중요하다. 

3) 알고리즘이란?
	- 용어 : 알고리즘, algorithm
	- 어떤 문제를 풀기 위한 절차/ 방법... 을 의미한다. 
	- 어떤 문제에 대해 특정한 '입력'을 넣으면, 원하는 '출력'을 얻을 수 있도록 만드는 프로그래밍 을 의미한다. 

	현실세계의 대표적 알고리즘..? -> 백종원의 레시피! ㅋㅋㅋ...
	-> 어떤 요리를 하려면 뭘 어떻게 해야할지, 알려주는 즉! 원하는 결과를 얻을 수 있도록 절차와 방법을 알려줌
	ex) 미역국을 만드는 방법. 

※ 좋은 알고리즘 이란 : 1) 얼마나 시간이 걸리느냐, 2) 얼마나 많은 데이터 공간을 차지하느냐. 
	이것에 따라 좋은 알고리즘인지 아닌지 갈린다.

4) 자료구조와 알고리즘이 중요한 이유
	=> 어떤 자료구조와 알고리즘을 쓰느냐에 따라 성능이 천지차이다!
	결국, 프로그래밍을 잘 할 수 있는 기술과 역량을 익히고, 검증할 수 있다. 


5) 자료구조/ 알고리즘, 그리고 파이썬
	- 어떤 언어로든 자료구조/알고리즘 익힐 수 있다.
		- 예전에는 무조건 C or C++로만 작성하도록 하는 경우가 많았다.
		- 최근에는 언어로 인한 제약/평가는 없어졌다.
	=> 가장 쉽고 빠르게 자료구조/알고리즘을 익힐 수 있는 언어가 파이썬이다.

	현업에서도 파이썬은 많이 사용된다.
